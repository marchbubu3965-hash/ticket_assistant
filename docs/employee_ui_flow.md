# 員工資料管理 UI Flow 規劃文件

## 目的

本文件用於定義「員工資料管理系統」的使用流程（UI Flow）與核心資料模型，作為後續實作 CLI / Desktop App / Web App 之依據。

---

## 一、員工資料模型（Data Model）

### Employee

| 欄位名稱        | 型別           | 說明         |
| ----------- | ------------ | ---------- |
| employee_id | string / int | 系統產生的唯一識別碼 |
| name        | string       | 員工姓名       |
| id_number   | string       | 身分證字號（唯一）  |
| department  | string       | 所屬單位       |
| role        | string       | 角色 / 職稱    |
| status      | enum         | 在職 / 停用    |
| created_at  | datetime     | 建立時間       |
| updated_at  | datetime     | 最後更新時間     |

---

## 二、主要畫面與操作流程

### 1️⃣ 員工列表畫面（Employee List View）

**功能**

* 顯示所有員工清單
* 顯示欄位：姓名、單位、角色、狀態
* 提供搜尋與篩選功能

**操作**

* 點擊「新增員工」
* 點擊員工列 → 進入編輯畫面

---

### 2️⃣ 新增員工流程（Create Employee）

**輸入欄位**

* 姓名（必填）
* 身分證字號（必填）
* 單位（必填）
* 角色（必填）

**流程**

1. 使用者填寫資料
2. 系統進行欄位驗證
3. 驗證成功 → 儲存資料
4. 顯示新增成功提示
5. 返回員工列表

---

### 3️⃣ 編輯員工資料（Edit Employee）

**可編輯項目**

* 單位
* 角色
* 狀態（在職 / 停用）

**流程**

1. 進入編輯畫面
2. 修改資料
3. 儲存變更
4. 顯示更新成功提示

---

### 4️⃣ 停用 / 啟用員工（Status Management）

**規則**

* 停用員工不可被指派為操作人
* 停用後仍保留資料（邏輯刪除）

---

## 三、驗證規則（Validation Rules）

### 身分證字號

* 格式必須合法（英文字母 + 9 碼數字）
* 不可重複

### 欄位驗證

* 必填欄位不可為空
* 單位 / 角色須存在於系統設定中

---

## 四、錯誤處理與提示

* 驗證失敗時顯示明確錯誤訊息
* 儲存失敗時顯示系統錯誤提示
